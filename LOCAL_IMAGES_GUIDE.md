# 📸 Руководство по использованию локальных AI-изображений

## ✅ Да, это возможно!

**Важно:** Локально загружаются **только AI-фотографии**.  
Реальные фото продолжают загружаться с внешних сервисов (Pexels/LoremFlickr) как и раньше.

Это решение даже лучше, чем внешние источники для AI-изображений, потому что:

- ✅ **Полный контроль** над качеством изображений
- ✅ **Нет зависимости** от внешних API
- ✅ **Быстрая загрузка** (изображения на вашем сервере)
- ✅ **Нет лимитов** на количество запросов
- ✅ **Гарантированная доступность**

---

## 📋 Пошаговая инструкция

### Шаг 1: Подготовка папок

Создайте структуру папок на сервере:

```bash
server/
  data/
    images/
      ai/          # ← Сюда загружайте ТОЛЬКО AI-изображения
```

**Примечание:** Папка `real/` не используется. Реальные фото продолжают загружаться автоматически с внешних сервисов (Pexels/LoremFlickr) при запуске `update-manifest.mjs`.

**На Windows (PowerShell):**
```powershell
New-Item -ItemType Directory -Path "server\data\images\ai" -Force
```

**На Linux/Mac:**
```bash
mkdir -p server/data/images/ai
```

**Примечание:** Папка `real/` не нужна. Реальные фото загружаются автоматически с внешних сервисов.

### Шаг 2: Загрузка изображений

Загрузите ваши AI-изображения в папку `server/data/images/ai/`.

**Рекомендации:**
- Форматы: `.jpg`, `.jpeg`, `.png`, `.webp`, `.gif`
- Размер: рекомендуется 600x450px или больше
- Имена файлов: можно использовать любые (например: `selfie-1.jpg`, `portrait-2.png`)

**Пример структуры:**
```
server/data/images/ai/
  ├── selfie-1.jpg
  ├── selfie-2.jpg
  ├── portrait-1.png
  ├── portrait-2.png
  ├── scene-1.jpg
  └── ... (1000-2000 файлов)
```

### Шаг 3: Добавление в манифест

Запустите скрипт для автоматического добавления всех изображений в `imageManifest.json`:

```bash
# Базовое использование (для localhost)
node scripts/add-local-images.mjs

# Для production (укажите ваш API домен)
node scripts/add-local-images.mjs --api-base=https://api.your-domain.com

# С дополнительными параметрами
node scripts/add-local-images.mjs \
  --dir=server/data/images/ai \
  --type=ai \
  --api-base=https://api.your-domain.com \
  --source="My AI Collection"
```

**Что делает скрипт:**
1. Сканирует папку `server/data/images/ai/`
2. Находит все изображения
3. Создает записи в `imageManifest.json` с правильными URL
4. Не дублирует уже существующие записи

### Шаг 4: Настройка API Base URL

**Важно!** Перед запуском скрипта убедитесь, что указан правильный `api-base`:

- **Development:** `http://localhost:3000`
- **Production:** `https://api.your-domain.com` (ваш реальный домен)

### Шаг 5: Перезапуск сервера

После добавления изображений в манифест перезапустите сервер:

```bash
# Если используете PM2
pm2 restart game-server

# Или просто
npm run dev
```

## 🔧 Как это работает

### 1. Хранение изображений

Изображения хранятся на сервере в папке `server/data/images/`:

```
server/data/images/
  └── ai/
      ├── photo-1.jpg
      ├── photo-2.jpg
      └── ... (1000-2000 файлов)
```

**Примечание:** Папка `real/` не используется. Реальные фото загружаются автоматически с внешних сервисов.

### 2. Раздача через Express

Сервер автоматически раздает изображения через Express static middleware:

```javascript
// server/index.js
app.use('/images', express.static(IMAGES_DIR));
```

**URL для доступа:**
- `https://api.your-domain.com/images/ai/photo-1.jpg`
- `https://api.your-domain.com/images/ai/photo-2.jpg`

### 3. Записи в манифесте

После запуска скрипта в `imageManifest.json` появятся записи:

```json
[
  {
    "id": "local-ai-selfie-1-0",
    "type": "ai",
    "url": "https://api.your-domain.com/images/ai/selfie-1.jpg",
    "source": {
      "name": "Local AI Images",
      "license": "Custom (Local Storage)"
    },
    "prompt": "selfie 1"
  },
  {
    "id": "local-ai-portrait-2-1",
    "type": "ai",
    "url": "https://api.your-domain.com/images/ai/portrait-2.png",
    "source": {
      "name": "Local AI Images",
      "license": "Custom (Local Storage)"
    },
    "prompt": "portrait 2"
  }
]
```

### 4. Использование в игре

Игра автоматически использует изображения из манифеста, включая локальные:

1. Сервер выбирает случайное изображение из манифеста
2. Возвращает URL клиенту
3. Клиент загружает изображение по этому URL
4. Изображение отображается в игре

## 📊 Примеры использования

### Пример 1: Добавление 1000 AI-изображений

```bash
# 1. Загрузите изображения в папку
# server/data/images/ai/photo-1.jpg
# server/data/images/ai/photo-2.jpg
# ... (1000 файлов)

# 2. Запустите скрипт
node scripts/add-local-images.mjs --api-base=https://api.your-game.com

# Вывод:
# ✅ Найдено изображений: 1000
# ✅ Добавлено новых: 1000
# ✅ Манифест сохранен

# 3. Перезапустите сервер
pm2 restart game-server
```

### Пример 2: Обновление реальных фотографий

**Примечание:** Реальные фото не загружаются локально. Они обновляются автоматически через скрипт `update-manifest.mjs` из внешних источников (Pexels/LoremFlickr).

```bash
# Чтобы обновить реальные фото из внешних источников:
npm run update-manifest
# или
node scripts/update-manifest.mjs --real=500
```

Локально загружаются **только AI-изображения** для лучшего контроля качества.

### Пример 3: Обновление существующих изображений

Если вы добавили новые изображения в папку, просто запустите скрипт снова:

```bash
node scripts/add-local-images.mjs --api-base=https://api.your-game.com

# Скрипт автоматически:
# - Пропустит уже добавленные изображения
# - Добавит только новые
```

## 🚀 Оптимизация

### CDN (рекомендуется для production)

Для лучшей производительности можно использовать CDN:

1. Загрузите изображения на CDN (Cloudflare, AWS S3, etc.)
2. Обновите URL в манифесте на CDN URL
3. Или используйте прокси через ваш сервер

### Сжатие изображений

Перед загрузкой рекомендуется сжать изображения:

```bash
# Используйте инструменты типа:
# - ImageOptim (Mac)
# - TinyPNG (онлайн)
# - imagemin (Node.js)
```

### Кэширование

Сервер уже настроен на кэширование изображений на 1 год:

```javascript
app.use('/images', express.static(IMAGES_DIR, {
  maxAge: '1y',
  etag: true,
}));
```

## ⚠️ Важные моменты

### 1. Размер папки

1000-2000 изображений могут занять:
- При ~200KB на изображение: **200-400 MB**
- При ~500KB на изображение: **500MB - 1GB**

Убедитесь, что на сервере достаточно места!

### 2. Backup

Регулярно делайте backup папки `server/data/images/`:

```bash
# Создать архив
tar -czf images-backup.tar.gz server/data/images/

# Или использовать rsync
rsync -av server/data/images/ /backup/images/
```

### 3. Git

**НЕ добавляйте** папку `server/data/images/` в Git! Добавьте в `.gitignore`:

```
server/data/images/
```

### 4. Права доступа

Убедитесь, что сервер имеет права на чтение файлов:

```bash
chmod -R 755 server/data/images/
```

## 📝 FAQ

**Q: Можно ли смешивать локальные и внешние изображения?**  
A: Да! Манифест может содержать:
- **Локальные AI-изображения** (из папки `server/data/images/ai/`)
- **Внешние AI-изображения** (из Pollinations/Lexica/Google - если настроены)
- **Внешние реальные фото** (из Pexels/LoremFlickr)

Игра автоматически использует все записи из манифеста.

**Q: Что если я запущу `update-manifest.mjs`?**  
A: Скрипт автоматически сохранит все локальные AI-изображения. Он обновит только внешние источники (реальные фото из Pexels и AI из внешних API), но не удалит ваши локальные AI-фото.

**Q: Нужно ли удалять старые записи из манифеста?**  
A: Не обязательно. Старые записи просто не будут использоваться, если их ID не попадет в случайную выборку.

**Q: Как обновить изображение?**  
A: Замените файл с тем же именем и перезапустите сервер. Или удалите старую запись из манифеста и добавьте новую.

**Q: Можно ли использовать подпапки?**  
A: Да! Структура может быть любой:
```
server/data/images/ai/
  ├── selfies/
  │   ├── photo-1.jpg
  │   └── photo-2.jpg
  ├── portraits/
  │   └── ...
  └── scenes/
      └── ...
```

Скрипт автоматически обработает все файлы, включая подпапки.
