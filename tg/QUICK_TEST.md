## 🧪 Тестирование

### Вариант 1: Локальное тестирование (без Telegram)

Откройте в браузере:
```
http://localhost:4173
```

Игра должна работать, но функции Telegram (CloudStorage, имя пользователя) работать не будут.

### Вариант 2: Полное тестирование через Telegram

#### Шаг 1: Установите ngrok

1. Скачайте с https://ngrok.com/download
2. Или через PowerShell (как администратор):
   ```powershell
   choco install ngrok
   ```

#### Шаг 2: Запустите ngrok

```bash
ngrok http 4173
```

Вы увидите что-то вроде:
```
Forwarding  https://abc123.ngrok.io -> http://localhost:4173
```

**Скопируйте HTTPS URL!** (например: `https://abc123.ngrok.io`)

#### Шаг 3: Создайте бота

1. Откройте Telegram
2. Найдите [@BotFather](https://t.me/botfather)
3. Отправьте `/newbot`
4. Следуйте инструкциям
5. **Сохраните токен бота**

#### Шаг 4: Создайте Web App

1. В BotFather отправьте `/newapp`
2. Выберите вашего бота
3. Заполните:
   - **Title**: `Нейросеть или реальность?`
   - **Description**: `Угадай, реальное фото или AI?`
   - **Photo**: (можно пропустить)
   - **Web App URL**: `https://ВАШ_NGROK_URL/client/`
   
   Например, если ngrok дал `https://abc123.ngrok.io`, то URL будет:
   ```
   https://abc123.ngrok.io/client/
   ```
   
   **Важно:** URL должен заканчиваться на `/client/`

#### Шаг 5: Протестируйте

1. Откройте вашего бота в Telegram
2. Отправьте `/start`
3. Должна появиться кнопка "🎮 Играть" (или подобная)
4. Нажмите на кнопку
5. Игра откроется в Telegram Web App!

## ⚠️ Важно

- ngrok URL меняется при каждом перезапуске (в бесплатной версии)
- Для production нужен постоянный домен
- CloudStorage работает только в официальных клиентах Telegram

## 🐛 Если что-то не работает

### Web App не открывается
- Проверьте, что ngrok запущен
- Проверьте URL в BotFather (должен заканчиваться на `/client/`)
- Убедитесь, что клиент запущен на порту 4173

### Ошибки в консоли
- Откройте DevTools в Telegram Desktop (`Ctrl+Shift+I`)
- Проверьте ошибки в консоли

### Backend недоступен
- Убедитесь, что backend запущен на порту 3000
- Проверьте `API_BASE` в `client/main.js`
