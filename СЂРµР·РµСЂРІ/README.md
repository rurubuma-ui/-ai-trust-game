# Игра "Нейросеть или Реальность" для Яндекс Игр

HTML5 игра, где игроки должны определить, является ли изображение реальной фотографией или создано нейросетью.

## Возможности

- **Одиночный режим**: Бесконечный тест на определение реальных и AI-изображений
- **Дуэли**: Онлайн-соревнование до 10 правильных ответов с таймером на 4 секунды
- **Система профилей**: Уровни, XP, достижения, рейтинг (ELO)
- **Ежедневные задания**: Ежедневные квесты и бонусы
- **Еженедельные вызовы**: Специальные задания на неделю с наградами
- **Мини-цели**: Краткосрочные вызовы с таймером
- **Мотивационные механики**: Streak-бонусы, множители XP, серии правильных ответов

## Установка и запуск

### Требования

- Node.js 18+
- npm или yarn

### Установка зависимостей

```bash
npm install
```

### Запуск сервера разработки

```bash
npm run dev:server
```

Сервер запустится на `http://localhost:3000`

### Запуск клиента разработки

В отдельном терминале:

```bash
npm run dev:client
```

Клиент будет доступен на `http://localhost:4173`

## Обновление заданий

После публикации игры на Яндекс Играх, задания можно обновлять без перевыпуска игры:

### 1. Ежедневные задания и мини-цели

Отредактируйте файл `server/data/quests.json`:

```json
{
  "version": 2,
  "lastUpdated": "2024-01-15T12:00:00Z",
  "dailyQuests": [...],
  "miniGoals": [...]
}
```

**Важно**: При каждом обновлении увеличивайте `version` на 1, чтобы клиенты загрузили новые задания.

### 2. Еженедельные задания (автоматическая ротация)

Еженедельные задания **автоматически меняются каждую неделю** из пула шаблонов!

- В файле `quests.json` создайте массив `weeklyChallengesPool` с 10-20 шаблонами заданий
- Сервер автоматически выбирает 3 случайных задания из пула каждую неделю
- Выбор детерминированный (на основе даты начала недели), поэтому все игроки видят одинаковые задания
- Задания меняются автоматически каждый понедельник

**Пример пула заданий:**
```json
{
  "weeklyChallengesPool": [
    {
      "id": "week_correct100",
      "title": "100 правильных за неделю",
      "description": "Дай 100 правильных ответов за неделю",
      "target": 100,
      "type": "correct",
      "xp": 200,
      "reward": "x1.5 XP на 1 час"
    },
    // ... еще 19 шаблонов
  ]
}
```

### 3. Мини-цели (автоматическая ротация)

Мини-цели **автоматически меняются каждый день** из пула шаблонов!

- В файле `quests.json` создайте массив `miniGoalsPool` с 10-20 шаблонами мини-целей
- Сервер автоматически выбирает 5 случайных мини-целей из пула каждый день
- Выбор детерминированный (на основе даты начала дня), поэтому все игроки видят одинаковые мини-цели
- Мини-цели меняются автоматически каждый день в полночь

**Пример пула мини-целей:**
```json
{
  "miniGoalsPool": [
    {
      "id": "mini_5in2min",
      "title": "5 за 2 минуты",
      "description": "Дай 5 правильных ответов за 2 минуты",
      "target": 5,
      "type": "correct",
      "timeLimit": 120000,
      "xp": 20
    },
    // ... еще 19 шаблонов
  ]
}
```

### 4. Автоматическая перезагрузка

Сервер автоматически отслеживает изменения в файле `quests.json` и перезагружает задания без перезапуска.

### 5. Как это работает

- **Ежедневные задания**: Обновляются вручную в файле, сбрасываются каждый день автоматически
- **Еженедельные задания**: Автоматически выбираются из пула каждую неделю (каждый понедельник)
- **Мини-цели**: Автоматически выбираются из пула каждый день (каждый день в полночь)
- Клиент загружает задания с сервера при инициализации и при открытии модального окна заданий
- Задания кэшируются на 5 минут для снижения нагрузки
- Если сервер недоступен, используются задания по умолчанию (встроенные в клиент)
- Версионирование позволяет обновлять задания только при наличии новой версии

### 4. API Endpoint

```
GET /api/quests
```

Возвращает:
```json
{
  "version": 1,
  "lastUpdated": "2024-01-01T00:00:00Z",
  "dailyQuests": [...],
  "weeklyChallenges": [...],
  "miniGoals": [...]
}
```

## Обновление базы изображений

Для обновления базы изображений используйте скрипт:

```bash
node scripts/update-manifest.mjs
```

По умолчанию загружает 500 реальных и 500 AI-изображений (всего 1000).

Для изменения количества:

```bash
node scripts/update-manifest.mjs --realCount 300 --aiCount 300
```

## API

### REST Endpoints

- `GET /health` — проверка работоспособности сервера
- `GET /api/questions/single` — новый вопрос для одиночного режима
- `POST /api/questions/single/:id/answer` — проверка ответа (`{ answer: "real" | "ai" }`), возвращает результат и следующий вопрос
- `GET /api/questions/batch` — пакетная загрузка вопросов для предзагрузки
- `GET /api/quests` — получение заданий (ежедневные, еженедельные, мини-цели)
- `GET /api/meta/sources` — агрегированная статистика по источникам

### WebSocket `/ws`

- `join-queue` / `leave-queue` — вход и выход из поиска соперника
- `leave-match` — досрочный выход из дуэли
- `answer` — отправка ответа в текущем раунде
- `client-ready` — сигнал готовности клиента (после предзагрузки)
- `image-ready` — сигнал загрузки изображения

## Подготовка билда для Яндекс Игр

1. Запакуйте содержимое `client/` (вместе с `index.html`) в ZIP-архив. При необходимости пропишите дополнительную инициализацию SDK Яндекс Игр в `main.js`.
2. В `client/index.html` уже подключён `https://yandex.ru/games/sdk/v2`; в продовой сборке SDK автоматически инициализируется, локально он просто не отработает.
3. Сервер WebSocket/REST потребуется развернуть отдельно (например, на Yandex Cloud Functions + Yandex Managed Service for Kubernetes, Railway, Render и т. п.). Не забудьте обновить `API_BASE`/`WS_BASE` в `client/main.js` на продакшен-адрес или прокидывать url через переменные окружения.
4. В консоли разработчика Яндекс Игр загрузите архив и укажите URL внешнего backend-сервиса.
