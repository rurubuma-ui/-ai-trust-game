# 📘 Пошаговая инструкция: Настройка Google Custom Search API

## 🎯 Что мы делаем

Настраиваем Google Custom Search API для автоматического поиска AI-изображений с Creative Commons лицензией по всему интернету.

---

## 📋 Шаг 1: Создание Google Cloud проекта

### 1.1. Откройте Google Cloud Console

Перейдите по ссылке: **https://console.cloud.google.com/**

> 💡 Если у вас нет аккаунта Google, создайте его (это бесплатно)

### 1.2. Создайте новый проект

1. В верхней части страницы нажмите на выпадающий список с названием проекта
2. Нажмите **"New Project"** (Новый проект)
3. Введите название проекта, например: `AI Images Search`
4. Нажмите **"Create"** (Создать)
5. Дождитесь создания проекта (обычно 10-30 секунд)

> ✅ **Готово!** Проект создан. Запомните его название.

---

## 📋 Шаг 2: Включение Custom Search API

### 2.1. Откройте библиотеку API

1. В левом меню найдите **"APIs & Services"** (APIs и сервисы)
2. Нажмите **"Library"** (Библиотека)

### 2.2. Найдите и включите Custom Search API

1. В поисковой строке введите: `Custom Search API`
2. Нажмите на **"Custom Search API"** в результатах
3. Нажмите кнопку **"Enable"** (Включить)
4. Дождитесь активации (обычно несколько секунд)

> ✅ **Готово!** API включен.

---

## 📋 Шаг 3: Создание API ключа

### 3.1. Откройте раздел Credentials

1. В левом меню найдите **"APIs & Services"** → **"Credentials"** (Учетные данные)
2. Или перейдите напрямую: **https://console.cloud.google.com/apis/credentials**

### 3.2. Создайте API ключ

1. Нажмите **"+ CREATE CREDENTIALS"** (Создать учетные данные)
2. Выберите **"API key"** (API ключ)
3. Ключ будет создан автоматически
4. **Скопируйте ключ** - он понадобится позже!

> ⚠️ **Важно:** Сохраните ключ в безопасном месте. Он выглядит примерно так:
> ```
> AIzaSyBxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
> ```

### 3.3. (Рекомендуется) Ограничьте ключ

Для безопасности ограничьте ключ только для Custom Search API:

1. Нажмите **"Restrict key"** (Ограничить ключ)
2. В разделе **"API restrictions"** выберите **"Restrict key"**
3. Выберите **"Custom Search API"**
4. Нажмите **"Save"** (Сохранить)

> ✅ **Готово!** API ключ создан и сохранен.

---

## 📋 Шаг 4: Создание Custom Search Engine (CSE)

### 4.1. Откройте Google Custom Search

Перейдите по ссылке: **https://programmablesearchengine.google.com/**

> 💡 Это отдельный сервис от Google Cloud, но использует тот же аккаунт

### 4.2. Создайте новый поисковый движок

1. Нажмите кнопку **"Add"** (Добавить) или **"Create a custom search engine"**
2. Заполните форму:
   - **Sites to search** (Сайты для поиска): введите `*` (звездочка)
     > ⚠️ **Важно:** Звездочка означает поиск по всему интернету
   - **Name** (Название): введите любое имя, например `AI Images Search`
   - **Language** (Язык): выберите `English` или `Russian`
3. Нажмите **"Create"** (Создать)

### 4.3. Получите Search Engine ID (CSE ID)

1. После создания вы попадете на страницу настроек
2. В левом меню нажмите **"Setup"** → **"Basics"**
3. Найдите поле **"Search engine ID"**
4. **Скопируйте ID** - он понадобится позже!

> ⚠️ **Важно:** Сохраните CSE ID. Он выглядит примерно так:
> ```
> 012345678901234567890:abcdefghijk
> ```

### 4.4. (Опционально) Включите поиск изображений

1. В левом меню нажмите **"Setup"** → **"Advanced"**
2. Найдите **"Image search"** и включите его
3. Нажмите **"Save"** (Сохранить)

> ✅ **Готово!** Custom Search Engine создан.

---

## 📋 Шаг 5: Настройка переменных окружения

### 5.1. Создайте файл .env

1. В корне проекта (там, где находится `package.json`) создайте файл `.env`
2. Скопируйте содержимое из `.env.example` (если он есть)

### 5.2. Заполните ключи

Откройте файл `.env` и вставьте ваши ключи:

```env
GOOGLE_API_KEY=AIzaSyBxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
GOOGLE_CSE_ID=012345678901234567890:abcdefghijk
```

> ⚠️ **Важно:** 
> - Замените значения на ваши реальные ключи
> - Не добавляйте кавычки вокруг значений
> - Не коммитьте `.env` файл в Git (он уже должен быть в `.gitignore`)

### 5.3. Для Windows PowerShell (если .env не работает)

Если скрипт не читает `.env` файл, установите переменные в PowerShell:

```powershell
$env:GOOGLE_API_KEY="ваш_ключ_здесь"
$env:GOOGLE_CSE_ID="ваш_cse_id_здесь"
```

Или установите системные переменные окружения через Панель управления Windows.

---

## 📋 Шаг 6: Установка dotenv (опционально, но рекомендуется)

Для автоматической загрузки `.env` файла установите `dotenv`:

```bash
npm install dotenv
```

Затем добавьте в начало `scripts/update-manifest.mjs`:

```javascript
import 'dotenv/config';
```

> 💡 Это позволит автоматически загружать переменные из `.env` файла

---

## 📋 Шаг 7: Проверка работы

### 7.1. Запустите тестовое обновление манифеста

```bash
node scripts/update-manifest.mjs --ai=50
```

### 7.2. Проверьте вывод

Вы должны увидеть в консоли:

```
[update-manifest] Google Custom Search API enabled for AI images (CC license filter)
[update-manifest] Using Google Custom Search API for AI images...
[update-manifest] Google Images: fetched X images using Y API requests
```

### 7.3. Если видите ошибки

**Ошибка: "API key not valid"**
- Проверьте, что ключ скопирован правильно (без пробелов)
- Убедитесь, что Custom Search API включен в Google Cloud Console

**Ошибка: "Custom Search Engine ID not found"**
- Проверьте, что CSE ID скопирован правильно
- Убедитесь, что Custom Search Engine создан и активен

**Ошибка: "Quota exceeded"**
- Вы превысили лимит 100 запросов/день
- Подождите до следующего дня или используйте платный план

## ✅ Готово!

Теперь Google Custom Search API настроен и будет автоматически использоваться для поиска AI-изображений с Creative Commons лицензией.

---

## 📊 Лимиты и ограничения

- **Бесплатный лимит:** 100 запросов/день
- **Изображений на запрос:** до 10
- **Максимум изображений в день:** ~1000 (при 100 запросах)
- **Платный план:** от $5 за 1000 дополнительных запросов

## 🔄 Что дальше?

1. Запустите полное обновление манифеста:
   ```bash
   node scripts/update-manifest.mjs --ai=500
   ```

   - Сохранит результаты в `server/data/imageManifest.json`

2. Убедитесь, что API включен в Google Cloud Console
3. Проверьте лимиты в Google Cloud Console → APIs & Services → Dashboard
